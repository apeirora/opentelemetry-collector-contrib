# otelauditcol

Minimal OpenTelemetry Collector distribution for secure audit-log ingestion and verification.

Built with [otel-collector-builder](https://github.com/open-telemetry/opentelemetry-collector/tree/main/cmd/builder) (OCB). Only the components required for the sync audit pipeline are included.

## Components

| Kind | Component |
| ---- | --------- |
| Receiver | `auditlogreceiver` |
| Processor | `certificatelogverify` |
| Exporter | `otlphttpexporter` |
| Extension | `redisstorageextension` |

Intentionally excluded: `batch`, `queuebatch`, `memorylimiter`, `debug`, and all other contrib/core components.

Pipeline:

```
auditlogreceiver (sync + WAL) → certificatelogverify → otlphttpexporter
```

See `receiver/auditlogreceiver/example-config.yaml` and `AUDITLOG_PIPELINE_PITFALLS.md` for operator rules.

## Security

- **Minimal attack surface** — OCB distro with four components only.
- **Image** — `scratch` base, non-root user `10001`, CA certs only from pinned Alpine stage.
- **Sync semantics** — no exporter `sending_queue`; inline bounded retry during the HTTP request.
- **Kubernetes** — hardened pod spec in `k8s/deployment.yaml` (`readOnlyRootFilesystem`, drop all capabilities, `runAsNonRoot`).

## Build locally

From the repository root:

```bash
make otelauditcol
./bin/otelauditcol_linux_amd64 --config=receiver/auditlogreceiver/example-config.yaml
```

Docker image:

```bash
make docker-otelauditcol
# or multi-stage build without a local Go toolchain:
make docker-otelauditcol-standalone
```

PowerShell deploy helper (build + optional kind/k8s):

```powershell
./deploy-auditlog-collector.ps1
./deploy-auditlog-collector.ps1 -UsePublishedImage
```

## Published image

Images are published to GitHub Container Registry by `.github/workflows/otelauditcol.yml` when changes land on `main` in the **apeirora** repository:

```text
ghcr.io/apeirora/opentelemetry-collector-contrib/otelauditcol:latest
ghcr.io/apeirora/opentelemetry-collector-contrib/otelauditcol:main
ghcr.io/apeirora/opentelemetry-collector-contrib/otelauditcol:sha-<commit>
ghcr.io/apeirora/opentelemetry-collector-contrib/otelauditcol:<semver>   # on version tags
```

Pull (package must be public, or authenticate to `ghcr.io`):

```bash
docker pull ghcr.io/apeirora/opentelemetry-collector-contrib/otelauditcol:latest
```

## Docker Compose demo

From this directory:

```bash
docker compose up --build
```

Optional: run the test backend on the host (`test-standalone` on port 9999) so exports succeed instead of returning 503.

```bash
cd ../../test-standalone && go run .
```

Collector listens on `https://localhost:4310/v1/audit` (TLS with dev test certificates mounted by compose).

## Kubernetes

Manifest: `k8s/deployment.yaml`

Default image:

```text
ghcr.io/apeirora/opentelemetry-collector-contrib/otelauditcol:latest
```

Replace Secret placeholders (`otelcol-audit-secrets`) before production use. The manifest expects a Redis service reachable at `redis:6379` in the same namespace.

## OCB regeneration

```bash
make genotelauditcol
```

Do not edit `components.go`, `main.go`, or `go.mod` in this directory by hand; they are generated by the builder.
